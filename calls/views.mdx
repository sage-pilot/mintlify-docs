Call views help teams move through voice work without creating a separate workflow for every status. Use views to answer the operational question in front of you: what is assigned to me, what needs pickup, what was missed, what did AI handle, and what needs review.

## Core views

| View | Use it for |
| --- | --- |
| All calls | Audit every call across the workspace. |
| My calls | Focus on calls assigned to you. |
| Incoming | Review calls initiated by customers. |
| Outgoing | Review calls placed by the team or automation. |
| Missed | Follow up on calls that were not answered. |
| Unassigned | Triage calls that do not yet have an owner. |
| Completed | Review finished calls and outcomes. |
| AI handled | Inspect calls handled by voice AI. |
| Scheduled | Prepare for upcoming scheduled calls. |
| Needs review | Review calls that require manager or QA attention. |

## Recommended workflow

1. Start from **Missed**, **Unassigned**, and **Needs review** when clearing operational queues.
2. Use **My calls** for individual agent follow-up.
3. Use **AI handled** to monitor automation quality and handoff behavior.
4. Use **Completed** when reviewing outcomes, recordings, transcripts, and QA notes.
