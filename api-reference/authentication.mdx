Sagepilot currently supports two customer-facing authentication models:

- API client v2 for analytics, exports, and upcoming platform APIs.
- Legacy workspace API key for existing WhatsApp template-send integrations only.

## Where to find your API details

You can find your credentials, channel IDs, and agent IDs in the Sagepilot dashboard:

**Settings** > **API Clients**

You must be a workspace admin to view or generate these credentials.

## API client v2

Use API client v2 for:

- Analytics API
- Ticket exports
- New platform API integrations

Create API client v2 credentials with this flow:

<Steps>
  <Step title="Open Sagepilot">
    Sign in to the Sagepilot workspace you want to connect.
  </Step>
  <Step title="Go to API Clients">
    Open **Settings** > **API Clients**.
  </Step>
  <Step title="Create an API client">
    Create a new API client and copy the `sp_...` token. It is shown once.
  </Step>
  <Step title="Send the token with each request">
    Pass the token as an `Authorization` bearer token or `X-API-Key` header.
  </Step>
</Steps>

Send the token in either of these forms:

```bash
Authorization: Bearer sp_<public_key_id>.<secret>
```

```bash
X-API-Key: sp_<public_key_id>.<secret>
```

<Warning>
Treat `sp_...` API client tokens as secrets. Do not commit them to source control or expose them in client-side code.
</Warning>

Use the platform host that matches your workspace region:

- Global region: `https://app.sagepilot.ai`
- Europe region: `https://eu.sagepilot.ai`

Workspace header behavior:

<ParamField header="X-Workspace-Id" type="string">
  Optional for API client v2 requests. If you send it, it must match the workspace bound to the API client.
</ParamField>

Typical access requirements:

- Analytics API: `analytics.query` scope plus dataset grants for analytics.
- Ticket exports: `tickets.export` scope plus `tickets.raw` dataset grant.

Recommended canonical dataset grants:

- `tickets`
- `messages`
- `outbound_messages`
- `csat`
- `workforce_activity`
- `tickets.raw`

## Legacy workspace API key

Use the legacy workspace API key only for existing WhatsApp template-send integrations.

Required headers:

```bash
Authorization: Bearer <legacy_workspace_api_key>
X-SP-Workspace-Id: <workspace_id>
```

<Warning>
This flow is for the legacy WhatsApp template-send route only. Do not use it for analytics, exports, or new integrations.
</Warning>

The workspace header is always required for the legacy flow.
