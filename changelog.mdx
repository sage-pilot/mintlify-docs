Use the changelog for customer-visible product updates, integration changes, API changes, and documentation updates.

## Latest entry template

Use one dated entry per release window. Keep the top summary short, then group details by product area.

```mdx
---
title: "May 29, 2026"
description: "Short summary of the release."
---

## What's new

- One-line customer-visible update.
- One-line workflow or integration update.
- One-line API or documentation update.

## Support inbox

### Clear feature title

Explain what changed, why it matters, and where users can find it.

## AI agents

### Clear feature title

Explain what changed for agent setup, review, quality, or handoff.

## Integrations

### Clear feature title

Explain what changed for channels, ecommerce, helpdesks, CRMs, or webhooks.

## API Reference

### Clear feature title

Explain new endpoints, changed schemas, authentication changes, or migration notes.
```

## Changelog rules

- Lead with what changed for the user.
- Group details by product area.
- Link to the relevant guide or API reference page when one exists.
- Avoid internal implementation details, raw PR titles, and engineering-only labels.
- Use exact dates for entries.
