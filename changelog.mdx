## New features

### <Badge color="green" size="sm" shape="pill">WhatsApp</Badge> Meta coexistence onboarding support

Teams can now onboard WhatsApp accounts with Meta coexistence support. This makes it easier to connect Sagepilot while continuing to work with existing Meta-connected WhatsApp setups.

### <Badge color="blue" size="sm" shape="pill">Integrations</Badge> Logistics integrations

Sagepilot now supports Fship, Shadowfax, Ekart, Shiprocket, Openleaf, and Shipway integrations. Teams can connect more delivery and fulfillment providers to improve shipment visibility inside customer and operations workflows.

### <Badge color="purple" size="sm" shape="pill">Search</Badge> Unified search

Unified search now brings together chats, calls, contacts, and social platform comments in one place. Teams can find customer conversations and activity faster without switching between separate modules.

### <Badge color="orange" size="sm" shape="pill">SDK</Badge> React Native SDK

Sagepilot now supports a React Native SDK for mobile app implementations. Teams building native mobile experiences can use Sagepilot messaging and customer workflows directly in their apps.

### <Badge color="surface" size="sm" shape="pill">MCP</Badge> Sage MCP

Sage MCP gives teams a structured way to connect Sagepilot context and supported actions into AI agent workflows. This helps agents work with workspace data through a more reliable integration layer.

## Enhancements

### <Badge color="blue" size="sm" shape="pill">RCS</Badge> Variable mapping support

RCS templates now support variable mapping, so teams can personalize messages with customer, campaign, journey, or order data before sending.

### <Badge color="green" size="sm" shape="pill">Analytics</Badge> Attribution details API

The analytics API now includes attribution details for journeys and campaigns, helping teams understand which sources, referrals, and campaign paths contributed to customer activity.

### <Badge color="purple" size="sm" shape="pill">Voice</Badge> Expanded call workflows

Voice calls now support back office tickets, tags, transcripts, template sending, and richer filters. This makes call operations easier to manage alongside the rest of the customer support workflow.

### <Badge color="orange" size="sm" shape="pill">Email</Badge> Version-based template duplication

Email templates can now be duplicated from specific versions. Teams can branch from a known working version when creating campaign variants or updating existing templates.

### <Badge color="green" size="sm" shape="pill">Website widget</Badge> Cart, CTA, and carousel support

The website widget now supports cart sync and cart management, custom CTAs, and carousel-style experiences. This helps teams create richer storefront and support flows inside the widget.

### <Badge color="blue" size="sm" shape="pill">Journeys</Badge> Custom API support for decision nodes

Journey decision nodes can now use custom API calls. Teams can route customers based on external data, business rules, eligibility checks, or operational state.

### <Badge color="purple" size="sm" shape="pill">Instagram</Badge> Carousel support

Instagram carousel content is now better supported across social engagement and conversation workflows, making it easier to handle multi-media posts and comment-to-DM interactions.
